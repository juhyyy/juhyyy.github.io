LLM + RAG Intent Analysis Pipeline

DX Education · 2025
Problem

Existing Korean intent classifier (~57% accuracy) frequently mis-routed user requests, breaking downstream automation and eroding trust.

Approach

Rebuilt intent taxonomy with domain experts, then designed a modular LangChain/LangGraph RAG pipeline with schema-validated LLM generations and robust error handling.

Impact

Improved intent accuracy to ~84%, reduced reasoning errors by 62.6%, and achieved 99% parsing stability — enabling reliable automation on top of the pipeline.

Virtual Brain Optimization (RL)

Neuromodulation Research · 2023–2024
Problem

Stimulation parameters for neuromodulation were tuned manually, making it slow and difficult to personalize across patients and scenarios.

Approach

Used DDPG reinforcement learning on The Virtual Brain simulations to learn policies that map brain states to optimized stimulation parameters, under clinical constraints.

Impact

Achieved an average 6.2% improvement in similarity to real brain signals (up to 15.87% for some subjects) with 3.6× faster runtime than grid search, making exploration of parameter space tractable.

EEG-based OSA Analysis (XAI)

Clinical AI · 2023
Problem

Clinicians needed interpretable EEG features to support AHI/SpO₂ prediction, not just black-box model outputs.

Approach

Engineered 29 time-series EEG features, trained ML regressors (XGBoost, SVM, RF), and added a SHAP-based explanation layer to surface channel-level importance.

Impact

Achieved 85–88% prediction accuracy while highlighting 6 key EEG channels, giving clinicians a transparent starting point for further investigation.

Approach

How I like to work across users, workflows, and engineering teams.

Start from the decision, not the model

I begin with the real decisions stakeholders need to make, the constraints they operate under, and what “better” looks like. From there, I work backwards to the data and technical choices.

Make complexity legible

I clarify assumptions, edge cases, and trade-offs early. I prefer explicit reasoning and documentation over “magic” so others can trust and extend the system.

Iterate with users in the loop

I like to ship small, focused workflows, gather feedback, and refine based on how people actually use the tool — keeping an eye on the metrics that matter for the organization.
